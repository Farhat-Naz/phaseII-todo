## Urdu Grammar & Pronunciation Guide

### Common Urdu Honorific Forms
- **آپ (Aap)** - Formal "you" (used throughout the app)
- **تم (Tum)** - Informal "you" (NOT used - app maintains formal tone)

### Common Verb Endings (Formal)
- **کریں (karein)** - "do" (command form, formal)
- **کرنا (karna)** - "to do" (infinitive)
- **ہے (hai)** - "is"
- **ہیں (hain)** - "are"

### Punctuation
- **۔** - Urdu period (replaces English .)
- **؟** - Urdu question mark (replaces English ?)
- **،** - Urdu comma

### Numbers
- Arabic-Indic numerals can be used, but Western numerals (0-9) are acceptable and used in the app for consistency

## Usage Examples in Code

### Basic Translation
```tsx
import { useTranslations } from 'next-intl';

export function LoginButton() {
  const t = useTranslations('auth');
  return <button>{t('loginButton')}</button>;
  // Renders: "Sign In" (en) or "داخل ہوں" (ur)
}
```

### With Pluralization
```tsx
const t = useTranslations('todos');
const minutes = 5;

<p>{t('minutesAgo', { count: minutes })}</p>
// Renders: "5 minutes ago" (en) or "5 منٹ پہلے" (ur)
```

### With Variable Interpolation
```tsx
const t = useTranslations('todos');
const charCount = 150;

<p>{t('charactersCount', { count: charCount })}</p>
// Renders: "150/2000 characters" (en) or "150/2000 حروف" (ur)
```

### Conditional RTL Styling
```tsx
import { useLocale } from 'next-intl';

export function TodoForm() {
  const locale = useLocale();
  const isRTL = locale === 'ur';

  return (
    <form dir={isRTL ? 'rtl' : 'ltr'} className={isRTL ? 'font-urdu' : ''}>
      {/* form content */}
    </form>
  );
}
```

## Translation Consistency Rules

1. **Formal Address:** Always use آپ (aap), never تم (tum)
2. **Technical Terms:** Keep romanized when no clear Urdu equivalent (e.g., "email", "password")
3. **Punctuation:** Use Urdu punctuation in Urdu strings
4. **Tone:** Professional, helpful, concise
5. **Action Verbs:** Use imperative form with کریں (karein) ending
6. **Questions:** End with ؟ (Urdu question mark)
7. **Consistency:** Same English term always maps to same Urdu term

## Common Translation Patterns

### Action Buttons
- **English Pattern:** Verb + Object (e.g., "Add Todo")
- **Urdu Pattern:** Object + Verb (e.g., "کام شامل کریں" - Todo Add)

### Validation Messages
- **English Pattern:** Field + "is required"
- **Urdu Pattern:** Field + "ضروری ہے" (zaroori hai)

### Status Messages
- **English Pattern:** Verb-ing + Object (e.g., "Loading todos...")
- **Urdu Pattern:** Object + Verb + "رہے ہیں" (e.g., "کام لوڈ ہو رہے ہیں")
